Smart Species Auto-Naming
In ReactionBuilderModal.tsx, addSpecies() hardcodes label = 'B' for reactants and 'S' for products — so clicking "Add Reactant" multiple times always appends another 'B'. Fix: compute the next available label from the current species array. Use uppercase sequential letters A–Z. For reactants: find the first letter in ['A','B','C','D','E','F'] not already present in species.map(s => s.label). For products: find the first letter in ['R','S','T','U','P','Q'] not already present. If all are taken, append a number suffix (e.g. "A2"). This ensures clicking "Add Reactant" three times gives A → B → C (if A is taken, B → C → D), and "Add Product" three times gives R → S → T. Also fix label uniqueness validation in validateReaction(): add a check that no two species share the same label, returning an error if duplicates found.
Fixes

Denbigh Reaction System
Implement the Denbigh scheme from Levenspiel Ch. 8: A→R (k₁), A→T (k₂), R→S (k₃), R→U (k₄). Rate equations: -rA = (k₁+k₂)·CA = k₁₂·CA, rR = k₁·CA − (k₃+k₄)·CR = k₁·CA − k₃₄·CR, rS = k₃·CR, rT = k₂·CA, rU = k₄·CR. New reactionMode: 'denbigh' in simulation.ts. New denbighPreset in reactionRegistry.ts using params.k as k₁, params.k2 as k₂, params.k3 as k₃, and new param params.k4: number (default 0.1) as k₄. Species: A, R, S, T, U. Five components. Add k4: number to SimulationParams (default 0.1, same migration checklist — sweepEngine, targetSolver, serializer). Add MODE dropdown option "Denbigh". Show k₃, k₄ inputs in settings panel when reactionMode === 'denbigh'. SpeciesProfile renders all 5 species. Levenspiel plot uses -rA = k₁₂·CA as the primary rate. Selectivity: define φR = CR_out / (Ca0 − CA_out) for instantaneous selectivity of R — display in the Analysis tab when denbigh mode is active. Test: CSTR, k1=1, k2=0.5, k3=2, k4=1, τ=1, Ca0=1 → verify CT_out = k₂/(k₁+k₂) · Xa (analytical: T yield = k2*Da/(k12*(1+k12*Da))).

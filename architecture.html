⚡ The two load-bearing contracts — define first, change no behavior in Slice 0
#4 Stream — species-centric, conversion is derived
interface Stream {
  // molar flow of each species [mol/s]
  F: Record<SpeciesId, number>;
  T: number;  // K
  P: number;  // Pa
}

// Xa is a DERIVED helper — never stored:
const getXa = (s: Stream, feed: Stream) =>
  1 - s.F['A'] / feed.F['A'];
#6 Unit Model — injected chemistry, no kinetics branching
type UnitModel = (
  inlets: Stream[],
  params: UnitParams,    // τ, vol, energy mode
  reactions: ReactionSet, // injected chemistry
  thermo: ThermoModel   // injected properties
) => {
  outlets: Stream[];
  profile: ProfilePoint[];  // for plots
  diagnostics: Diagnostics; // converged?, warnings
}
The operating principle: The unit evaluates an injected ReactionSet — it never does if (kinetics === 'first-order'). This collapses reactorSolvers + thermalSolvers + dynamicSolvers into two functions (CSTR, PFR). Adding reversible reactions, a 4th species, or user-defined expressions requires zero changes to unit code. Emergence from composition, not from growing the switch statement.

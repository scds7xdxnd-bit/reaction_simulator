⚠ Known bugs entering Phase 3
Parallel network returns null solveNetwork returns null for diamond topologies (Feed→Splitter→{A,B}→Mixer→Product). Status bar shows "Connect Feed → Units → Product". Root cause is in forwardPass: splitter only sets edge streams when sourceHandle === 'out-top'/'out-bot' — if user connects via the generic handle, no stream is written. Downstream nodes fall back to fresh params. Compounded by a potential runtime crash in topoSort when stale edges from copy-paste reference non-existent node IDs. Fix: (1) robust splitter stream-set by edge order, not handle name; (2) guard stale edges in topoSort.
Duplicate node labels after copy-paste Copy-pasted nodes keep the original label ("CSTR-2" appears twice). No store counter increment on paste, so new nodes don't get incremented names. Cosmetic only but confusing. Fix: increment cstrCount / pfrCount in the paste action.
Xa: — on nodes not in flow path Reactors not reachable feed→product show "Xa: —" without explanation. Nodes in an island show no diagnostic message. Could mislead user into thinking the reactor ran. Fix: topology validator should show per-node warnings; nodes outside the flow path should show a distinct visual state (orange border, tooltip).

Pressure Drop — PFR
🔑 Momentum balance adds a 3rd ODE to the PFR system — architecture decision needed
Add optional Ergun equation to the PFR ODE system: dP/dz = -G/ρ·(1-φ)/φ³ · (150(1-φ)μ/Dp² + 1.75G/Dp). New PFR node toggle "Pressure Drop" with inputs: catalyst particle size Dp, void fraction φ, inlet pressure P0. The ODE becomes [Ca, Cr, Cs, T, P] (5-vector). The rate law is multiplied by (P/P0)ⁿ for gas-phase. Architecture: pfrModel already integrates a vector ODE — extend the state vector from [Ca,Cr,Cs,T] to [Ca,Cr,Cs,T,P] when pressure-drop mode is enabled.
Enables
constant-pressure PFR only → packed bed PFR · Ergun ΔP · optimal bed length design
Later
12
Gas-Phase Reactions
Add volumetric expansion for gas-phase reactions. New parameter ε (expansion factor, dimensionless). Concentration becomes Ca = Ca0·(1-Xa)/(1+ε·Xa). New kinetics mode "Gas-phase 1st order" alongside the existing modes. For CSTR: modified CSTR design equation. For PFR: modify the ODE dFa/dV = -ra form (Fa-based, not Ca-based). Architecture: add gasPhase: boolean and epsilon: number to SimulationParams; rate laws receive the gas-corrected concentration.
Enables
liquid-phase reactions only → gas-phase with volumetric expansion · Fa-based design

The existing buildOperatingDiagram already computes the G(T) and R(T) curves — extend it to count and return all intersection points.

🔑 Architectural decisions to get right upfront
Splitter stream assignment: handle-name vs. edge-order
✗ Continue matching sourceHandle === 'out-top'/'out-bot' — breaks when user connects via any other handle
✓ Fall back to edge-index: outEdges[0] gets α fraction, outEdges[1] gets (1−α). Keep handle-name check as the primary path, add index fallback. Guard stale edges (source/target not in nodeMap) before topoSort.

The splitter has always had two distinct source handles for a reason (top/bot fraction assignment). But the index fallback makes it robust to ReactFlow handle connection behaviour variance.

Per-node validation: pure vs. context
✗ Import validation results inside each node component — breaks purity test for validation.ts
✓ validation.ts stays pure (returns nodeIssues: Map<id, issue[]>). A new ValidationContext (React context, not Zustand) distributes the map to nodes. Nodes read from context, not store.

Validation context is the right seam: topology-local information flows naturally as React context, not global Zustand state.

PFR ODE state vector extension for pressure drop
✗ Add a separate pressure-drop ODE pass after the main integration — doubles the integration cost and makes T/P coupling incorrect
✓ Extend the RK4 state vector from [Ca, Cr, Cs, T] to [Ca, Cr, Cs, T, P] when pressure-drop is enabled. The existing rk4Step already handles arbitrary-length vectors. Add a pressureDrop flag to UnitParams.

The correct approach: T and P are coupled (temperature affects viscosity, which affects ΔP). They must be integrated together in one RK4 step.

Dark mode strategy: Tailwind class vs. CSS variables
✗ Use prefers-color-scheme media query in CSS only — can't be overridden by the user's manual toggle
✓ Tailwind darkMode: 'class' strategy. A themeSlice in Zustand holds 'light' | 'dark' | 'system'. On mount, check matchMedia; on toggle, flip the dark class on <html>. Persisted in localStorage.

Class-based dark mode is the only approach that supports both system-preference detection AND user override with a single toggle.

Phase 3 purity rule: The pure-function invariant is preserved. New pure files added in P4 (pressureDropModel.ts, gasPhaseFactor.ts, steadyStateMapper.ts) get added to PURE_FILES in purity.test.ts. The validation map return from validation.ts is type-only; the ValidationContext React context lives in a separate src/context/ file. Tests target: 21+ tests (current 15 + 2 golden for parallel network + 4 for new pure math files).
